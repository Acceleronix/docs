---
title: 'Get Product TSL Model'
openapi: 'GET /v2/quectsl/openapi/product/export/tslFile'
---

This endpoint retrieves the Thing Specification Language (TSL) model for a product, which defines the properties, services, and events that devices of this product type support.

## Authentication Required

This endpoint requires a valid JWT token obtained through the authentication API. Include the token in the Authorization header:

```
Authorization: YOUR_FULL_ACCESS_TOKEN
```

**⚠️ Important**: Use the complete `access_token` value from the authentication response, including the "QJWT" prefix if present. Do NOT add "Bearer" prefix - the token already contains the necessary authentication type.

**Example**:
- If your token is: `QJWT eyJhbGciOiJIUzI1NiJ9...`
- Use: `Authorization: QJWT eyJhbGciOiJIUzI1NiJ9...`
- Do NOT use: `Authorization: Bearer QJWT eyJhbGciOiJIUzI1NiJ9...`

## Example Usage

Get TSL model for a specific product:

```bash
curl -X GET "https://iot-api.acceleronix.io/v2/quectsl/openapi/product/export/tslFile?productKey=YOUR_PRODUCT_KEY&language=EN" \
  -H "Authorization: YOUR_COMPLETE_ACCESS_TOKEN" \
  -H "Content-Type: application/json"
```

**Troubleshooting 401 "token not exists" error**:
1. **Use complete token**: Include "QJWT" prefix without "Bearer"
2. **Check token expiry**: Token expires in 24 hours (86400 seconds)  
3. **Verify product authorization**: Ensure your SaaS app has access to the specified product
4. **Check product key**: Verify the productKey exists and is accessible
